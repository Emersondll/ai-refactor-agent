### Spec

A **pure data holder** is a `class` whose every method is one of: a getter (`public T getX() { return field; }`), a setter (`public void setX(T v) { this.field = v; }`), a constructor, or `equals`/`hashCode`/`toString`. Any method with other logic disqualifies the class. `@Document`/`@Entity`/`@Table` are common but NOT required signals.

`MerchantDocument`, `TransactionDocument`, `BalanceDocument` in the target repo are pure data holders.

For such a class, the unit test is fully mechanical and must be generated by a Python template — never the LLM.

### Module API

```python
def is_pure_data_holder(code: str) -> bool
def generate_data_holder_test(code: str, test_class_name: str, test_package: str) -> str | None
```

`generate_data_holder_test` returns `None` (caller falls back to the LLM) when:
- the class is not a pure data holder, OR
- any constructor parameter / field has a type not in the supported sample-value table.

### Supported field types — sample value table

| Java type | import needed | sample A | sample B |
|-----------|--------------|----------|----------|
| `String` | — | `"sampleA"` | `"sampleB"` |
| `BigDecimal` | `java.math.BigDecimal` | `new BigDecimal("100.00")` | `new BigDecimal("250.50")` |
| `BigInteger` | `java.math.BigInteger` | `BigInteger.valueOf(100)` | `BigInteger.valueOf(250)` |
| `Timestamp` | `java.sql.Timestamp` | `Timestamp.valueOf("2023-01-15 10:00:00")` | `Timestamp.valueOf("2024-06-20 14:30:00")` |
| `long` / `Long` | — | `1L` | `2L` |
| `int` / `Integer` | — | `1` | `2` |
| `double` / `Double` | — | `1.0` | `2.0` |
| `boolean` / `Boolean` | — | `true` | `false` |
| `LocalDate` | `java.time.LocalDate` | `LocalDate.of(2023, 1, 15)` | `LocalDate.of(2024, 6, 20)` |
| `LocalDateTime` | `java.time.LocalDateTime` | `LocalDateTime.of(2023, 1, 15, 10, 0)` | `LocalDateTime.of(2024, 6, 20, 14, 30)` |

Any other type (enums, project domain classes, collections) → unsupported → return `None`.

### Generated test shape

```
package <test_package>;

import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.*;
<sorted unique type imports>
import <prod_package>.<ClassName>;

class <ClassName>Test {

    @Test
    void constructor_shouldInitializeAllFields() {
        <ClassName> obj = new <ClassName>(<sample A for each ctor param, in order>);
        assertEquals(<sample A>, obj.get<Field>());   // one per ctor param that has a getter
        ...
    }

    @Test
    void set<Field>_get<Field>_roundTrip() {           // one @Test per setter that has a matching getter
        <ClassName> obj = new <ClassName>(<sample A for all ctor params>);
        obj.set<Field>(<sample B>);
        assertEquals(<sample B>, obj.get<Field>());
    }
}
```

Rules for the generated code:
- Use the all-args constructor. If the only constructor is no-arg, or if constructor params do not 1:1 map (by name) to fields with getters, return `None`.
- For `boolean` fields the getter may be `isX()` — handle both `get`/`is` prefixes.
- Class declaration is package-private `class <Name>Test` (no `public`) — matches existing generated tests.
- The test class name MUST equal `test_class_name` (without `.java`).

```python
def commit_single_file(repo_path: str, file_abs_path: str, message: str) -> bool:
    """Commit a single file locally (no push). Crash-safety for long test runs."""
    import os as _os
    rel = _os.path.relpath(file_abs_path, repo_path)
    run_cmd(f'git add "{rel}"', cwd=repo_path)
    code, _, _ = run_cmd(f'git commit -m "{message}" -- "{rel}"', cwd=repo_path)
    return code == 0
```
Local commit only — the final `commit_and_push` at end of run still pushes everything.

## Task P4: Surgical line-patch repair

**Files:**
- Modify: `java/refactor.py` — add `_try_surgical_patch()`, hook into the repair loop (~line 1673)

### Spec

For assertion failures where the exact one-line transformation is known, apply the patch in Python instead of regenerating via the LLM.

`_try_surgical_patch(test_code: str, maven_output: str) -> str | None`:
1. Parse `maven_output` for `<TestClass>.<method>:<lineNumber>` AND the `expected: <X> but was: <Y>` values.
2. If found, locate that line in `test_code`.
3. Apply the transformation:
   - `expected=<null>`, actual non-null → `assertNull(expr)` → `assertEquals("<actual>", expr)`
   - `actual=<null>`, expected non-null → `assertEquals(X, expr)` → `assertNull(expr)`
   - both non-null, simple value mismatch → replace `<X>` literal with `<Y>` on that line
4. Return the patched full file, or `None` if the line doesn't match an expected shape (caller falls back to the LLM).

### Steps

- [ ] **Step 3:** Implement `_try_surgical_patch` in `java/refactor.py`.
- [ ] **Step 4:** Run — verify pass.
- [ ] **Step 5:** Hook into the repair loop: in the `if not success:` branch (~line 1673), BEFORE building `error_msg` / calling `call_ai_with_correction`, try `_try_surgical_patch(test_code, combined_out)`. If it returns patched code, `write_file`, re-run `maven_test_with_coverage`, and `continue` the repair loop without an LLM call. Only fall through to the LLM if the patch returns `None`.
