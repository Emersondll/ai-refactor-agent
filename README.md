## Pipeline Architecture

```
main.py
  ├── HEALTH_CHECK                 → maven_test (validates initial state)
  ├── AUDIT_COVERAGE               → java/refactor.py · generate_tests()
  │     ├── P0: data holder?       → java/data_holder_test_gen.py (Python template, no LLM)
  │     │                                 ├── success → commit_single_file (P2) + continue
  │     │                                 └── failure → fallback to LLM
  │     └── LLM path               → call_ai (fixed seed P1) → repair loop
  │                                       ├── P4: _try_surgical_patch (Python, before LLM)
  │                                       └── call_ai_with_correction (LLM fallback)
  │
  ├── Phases 01–14 (sequential loop via phases/configs/*.yml)
  │     ├── tool: community        → java/community_runner.py
  │     ├── tool: llm              → java/llm_runner.py
  │     │                               ├── method_level: true  → java/method_runner.py (_run_method_level)
  │     │                               ├── class_level:  true  → java/method_runner.py (_run_class_level)
  │     │                               └── (no flag)           → llm_runner loop per file
  │     ├── tool: flow             → java/flow_runner.py
  │     └── tool: flow-dry         → java/flow_runner.py (dry_check)
  │
  ├── AUDIT_COVERAGE_POST_DIP (S5) → java/refactor.py (second pass — classes released by solid-dip)
  ├── SANITIZATION                 → java/sanitizer.py (dead imports, inactive code via regex)
  ├── JAVADOC                      → java/javadoc_runner.py (Javadoc on public methods)
  ├── FINAL_VALIDATION             → maven_test + JaCoCo (final coverage)
  ├── REPORT                       → java/report_runner.py (Markdown report per class)
  └── COMMIT_PUSH                  → git_utils/repo.py (branch refactor/ai-agent-automation, final push)
```

## Method-by-Method Refactoring (Phases 09, 10, 12)

```
For each Java file (excl. tests):
  │
  ├── cache.is_phase_done?       → skip
  ├── _is_structural_type?       → skip (FILE_SKIPPED — orange in dashboard)
  │     record / interface / @Entity / @Document / DTO
  │
  └── For each method:
        ├── cache.is_method_done?           → skip
        ├── detect_fn(method.body)?         → pattern absent → mark done, skip
        │
        ├── build_method_context()
        │     └── class skeleton (other methods as signatures)
        │         + full target method
        │
        ├── call_ai(method.full_text, dep_context=skeleton)
        │     └── LLM returns ONLY the refactored method
        │
        ├── extract_method_from_response()
        ├── merge_method(current_code, original, new)
        ├── mvn compile -q
        │     ├── OK  → mark_method_done + FILE_ACCEPTED
        │     └── ERR → write_file(previous_code) + FILE_REVERTED
        └── mark_phase_done(file) after all methods evaluated
```

## Full-Class Refactoring (Phase 11 — solid-dip)

```
For each Java file:
  ├── _is_structural_type? → skip
  ├── skip_compression: true (flag in yml)
  │     → sends ORIGINAL full code to LLM (full dependency context)
  ├── call_ai(full_class)
  │     └── LLM returns entire class (only constructor injection added)
  ├── mvn compile -q
  │     ├── OK  → write + mark_phase_done + FILE_ACCEPTED
  │     └── ERR → repair loop (A1: up to MAX_RETRIES attempts with call_ai_with_correction)
  │               → exhausted: git checkout -- file + FILE_REVERTED
  └── mark_phase_done
```

## Pipeline Determinism (P0–P4)

Five mechanisms eliminate the whack-a-mole of reactive fixes on non-deterministic generation:

| ID | Where | What |
|----|------|-------|
| **P0** | `java/data_holder_test_gen.py` (new) + `generate_tests()` | Deterministic test generator for pure data holders (`@Document`/`@Entity`/POJO with only fields + ctor + getters/setters). `is_pure_data_holder()` requires literal `return field;` and `this.field = param;` (ternary/method/concat fail). Supported types: `String`, `BigDecimal`, `BigInteger`, `Timestamp` (SQL format), `Long`/`Integer`/`Double`/`Boolean`, `LocalDate`, `LocalDateTime`. Other types → `None` → LLM fallback. Bypasses LLM completely — zero repair loop, fully reproducible. |
| **P1** | `config.py` + `ai/model.py · call_model()` | `OLLAMA_SEED=42` (override via env) injected into `options`. Same prompt → same output run-to-run. Fixes "stick" instead of re-rolling. Repair loop is not affected: changes the prompt on each attempt. |
| **P2** | `git_utils/repo.py · commit_single_file()` | Local commit per accepted file (no push) at both acceptance points (deterministic P0 and LLM). Interrupted run does not lose work — next run sees the commits and skips. `commit_and_push` at the end still `push`es everything. |
| **P3** | `java/refactor.py · _build_active_rules()` (extracted helper) | Consolidated 12+ `### ... (MANDATORY)` blocks from `active_rules`. Merged `IMPORTS PRESENT` + `SELF-IMPORT` + `IMPORT PROHIBITION` into a single `### IMPORTS`. Each conditional block compressed to ≤4 lines. Mandatory declarations stated once at the top. Regression test (`tests/java/test_active_rules_preservation.py`) verifies 18 key phrases preserved from all previous fixes. |
| **P4** | `java/refactor.py · _try_surgical_patch()` | One-line Python patch for deterministic assertion errors: `assertNull(expr)` ↔ `assertEquals("X", expr)` (S1/S4) and simple value swap (G1). Attempted BEFORE `call_ai_with_correction` in the repair loop. If the patch passes the build → `continue`; otherwise LLM fallback. Window ±3 lines to tolerate off-by-N from Maven. |

> **Why it matters:** the LLM (Ollama, 7–9B) is non-deterministic. Without this, each run regenerated tests with new variations that fell into new failures — error handlers were added endlessly. P0 eliminates the LLM for mechanical tasks. P1 makes what remains reproducible. P2 gives crash-safety. P3 reduces prompt noise. P4 avoids full-file regeneration for one-line edits.

## Javadoc Phase (post-sanitization)

```
For each production Java file:
  ├── _all_public_methods_documented? → FILE_SKIPPED (already_documented)
  ├── call_ai(full_class, javadoc rules)        ← MODEL_DOC (qwen2.5-coder:7b, code-aware)
  │     └── LLM adds /** */ where missing — without touching method bodies
  ├── _strip_comments(new_code) == _strip_comments(original)?
  │     ├── NO  → J1: retry with MODEL_STRUCT (qwen2.5-coder:7b) + critique prompt
  │     │         ├── retry accepted?   → continue to compile
  │     │         └── retry rejected    → FILE_SKIPPED (code_structure_changed)
  │     └── YES → continue to compile
  ├── mvn compile -q
  │     ├── OK  → FILE_ACCEPTED
  │     └── ERR → write_file(original) + FILE_REVERTED
  └── continue to next file
```

## Refactoring Report (post-final-validation)

```
report_runner.run_report()
  ├── Reads execution.jsonl — isolates current session (since GIT_BRANCH_CREATED)
  ├── Groups events by file:
  │     accepted  → phases where FILE_ACCEPTED
  │     skipped   → phases where FILE_SKIPPED (with reason)
  │     reverted  → phases where FILE_REVERTED
  ├── Serializes compact summary
  ├── Calls LLM (Claude → local model fallback)
  │     → generates narrative Markdown report
  │     → sections: Overview, Coverage, Modified Classes, Skipped, Reverted
  ├── Pure Python fallback if LLM unavailable
  ├── Saves to logs/refactoring_report.md
  └── Saves to REFACTORING_REPORT.md in the target repo root (committed)
```

**Visualization:** `http://localhost:8000/report.html` (link on dashboard) — Markdown rendering with dark theme, auto-refresh every 30s.

## LLM Skills

10 skills in `skills/<name>/SKILL.md` at the project root. Loaded via `core.utils.load_skill(name, section="...")` and injected as prompt instruction into local LLMs (Ollama). They are Markdown prompt snippets — NOT Claude sub-agents.

### Execution Map

| Moment | Skill | Where loaded | Trigger |
|---------|-------|-----------------|---------|
| Every refactoring call | `java-refactor-context` | `ai/prompt.py → _build_task()` | always — base instruction for any LLM phase |
| `AUDIT_COVERAGE` (pre-refactoring) | `java-tdd-unit-test` | `java/refactor.py → generate_tests()` | JaCoCo coverage < 90% |
| Phase 09 | `java-guard-clauses` | `method_runner._run_method_level()` | detects ≥ 3 levels of nested `if` |
| Phase 10 | `java-method-extraction` | `method_runner._run_method_level()` | detects method > 30 lines |
| Phase 11 | `java-solid-dip` | `method_runner._run_class_level()` | detects hardcoded `new ConcreteClass()` |
| Phase 12 | `java-controller-lean` | `method_runner._run_method_level()` | detects business logic in `@RestController` |
| Phase 13 | `java-flow-refactor` | `flow_runner.py` | all endpoints mapped by `flow_mapper` |
| Phase 14 | `java-dry-extraction` | `flow_runner.dry_check()` | file groups with repeated patterns |
| `AUDIT_COVERAGE_POST_DIP` (S5) | `java-tdd-unit-test` | `main.py → generate_tests()` | after phases 01–14 — classes with field injection converted by solid-dip |
| Phase 15 | `java-javadoc` | `javadoc_runner.py` | public method without `/** */` detected |
| Repair loop (phases 09–14) | `java-repair-guide` | `java/refactor.py → call_ai_with_correction()` | Maven compilation fails after LLM generation |

### LLM Phase Parameters (09–12)

| Skill | yml | method_level | class_level | skip_compression | detect_pattern |
|-------|-----|:---:|:---:|:---:|----------------|
| `java-guard-clauses` | `09_guard_clauses.yml` | ✓ | — | — | `nested_if` |
| `java-method-extraction` | `10_method_extraction.yml` | ✓ | — | — | `long_method` |
| `java-solid-dip` | `11_solid_dip.yml` | — | ✓ | ✓ | `concrete_new` |
| `java-controller-lean` | `12_controller_lean.yml` | ✓ | — | — | `controller_logic` |

> **solid-dip** uses `skip_compression: true` — sends the entire class to the LLM (without method compression) to preserve full dependency context. Absolute prohibitions: never create interfaces, never change signatures, never add logic — only replace `new ConcreteClass()` with constructor injection.

> **java-refactor-context** is the base context skill: every LLM refactoring call (phases 09–14 and repair loop) starts with this skill's instructions as the global rules system, before the phase-specific rules.

### Types Ignored in All LLM Phases

`_is_structural_type()` in `llm_runner.py` detects and emits `FILE_SKIPPED` (neon orange on dashboard) for: records, interfaces, `@Entity`, `@Document`, `@Table`, `*Dto.java`, `*DTO.java`, `*Request.java`, `*Response.java`, any file under `/dto/`. Zero tokens consumed on these types.

## Integrated Quality Techniques

- **Reviewer Pattern**: post-phase diff evaluated by `llm_reviewer.py` with skill-specific criteria → APPROVE/REJECT/SKIP
- **TDD Unit Test Skill**: autonomously generates tests before refactoring, preserving current behavior; 90% coverage gate
- **JaCoCo Guardrail**: minimum coverage ≥ 90%; regression alert if coverage drops > 1pp after refactoring
- **Repair Loop (A1)**: compile failure → captures exact error → `call_ai_with_correction()` → up to `MAX_RETRIES` attempts with `_categorize_build_error()` identifying type (String→BigDecimal, invalid constructor, missing symbol, etc.)
- **Structural Type Skip**: `_is_structural_type()` detects records, interfaces, @Entity, @Document, DTOs — zero tokens wasted
- **Controller Guard (C1)**: `controller-lean` only runs on `@RestController` classes; ServiceImpl skipped automatically
- **Agent Gitignore**: `_ensure_agent_gitignore()` injects `.refactor_cache/` and `.rag_store/` into the target repo's `.gitignore` before each commit — prevents internal files from appearing in history
- **Deferred Field Injection (M7)**: detects classes with `@Autowired` on field without constructor — defers test generation until solid-dip converts to constructor injection; `_dip_prefiltered` detects when solid-dip will never process it and uses `@InjectMocks` immediately
- **Post-DIP Coverage (S5)**: second pass of `generate_tests()` after all phases — classes that M7 deferred and that solid-dip converted now receive tests with injected constructor; classes already covered (≥ 90%) skipped by M8 at no cost
- **Deterministic Import Injection (S1/F1)**: `_auto_inject_missing_imports()` deterministically injects missing imports after generation and after each repair; uses `_s1_imports = prod_imports + [self_import]` — the self-import (e.g.: `import com.caju.transactionauthorizer.document.MerchantDocument;`) is derived from the package + production file name, covering the case where the class under test does not import itself
- **Import-Aware Repair (S2/S4)**: `_categorize_build_error()` includes exact import when reporting missing symbol; C1 links import to corresponding `@Mock` in the prompt, reducing type hallucination in test generation
- **Package Guard (F2)**: after C2 validates class name in the repair loop, new package check — if the LLM changed `package` to `com.example.model` (or any value differing from `_test_pkg`), injects `PACKAGE CRITICAL ERROR` into `combined_out` and forces a new attempt without writing the corrupted file to disk
- **Surgical Assertion Fix (G1/F4)**: `_categorize_build_error()` extracts `expected: <X> but was: <Y>` from Maven output and returns a surgical instruction — "find the assertion containing X and replace with Y, ONE LINE CHANGE only" — prevents the LLM from rewriting the entire test when correcting a wrong value, which caused compile errors in the next repair; F4 (null vs "") is a special case of the same handler
- **Javadoc Retry (J1)**: when `neural-chat:7b` (MODEL_DOC) modifies code beyond comments, `javadoc_runner.py` runs a retry with `MODEL_STRUCT` (qwen2.5-coder:7b, temp=0.05) before rejecting — reduces from ~11 to ~2–3 `code_structure_changed` failures per cycle
- **Constructor Call Hint (C1)**: `_extract_simplified_header()` in `context.py` generates `// CONSTRUCTOR CALL: new Foo(a, b)` for regular classes with explicit constructor (not only records) — the LLM uses this hint when instantiating objects in tests, eliminating `new Foo()` no-arg calls on `@Document`/`@Entity`
- **Repair-to-DepContext Bridge (R1)**: `_categorize_build_error()` in the RECORD/CONSTRUCTOR ERROR now explicitly instructs: "Look in DEPENDENCY CONTEXT for `// CONSTRUCTOR CALL:` and copy that EXACT call" — closes the loop between the dep_context hint (C1) and the error repair (D/R1)
- **Test Timeout Margin (T1)**: `TIMEOUT_TEST` increased from 300s to 420s — the real cost per file is ~50s (building the prompt KV cache in Ollama) + ~250s (generation), totaling ~300s. With the previous limit, attempt 1 expired systematically for every new file; attempt 2 always succeeded because the KV cache was already computed. With 420s, attempt 1 passes with margin even for larger prompts or hardware variation
- **Permanent Skip Expiry (P1)**: `_AUTO_EXPIRE_STACK_PATTERNS = ["com.example"]` in `refactor.py`. `FailedFilesTracker.reset()` automatically discards `permanent_skip` entries whose `stack_trace` contains a bug pattern already fixed in the pipeline — e.g.: package hallucination `com.example.*` fixed by F2 (Package Guard). In practice, TransactionCodeModelTest and TransactionControllerTest that were permanently blocked before F2 return to the queue on the next run without manual intervention. `clear_permanent_skips(file_path=None)` available for manual removal (specific file or all)
- **JUnit Static Import Detection (P2)**: `_JUNIT_ASSERTION_METHODS` frozenset with 18 JUnit 5 methods (`assertTrue`, `assertEquals`, `assertNull`, etc.). `_categorize_build_error()` in the `METHOD ERROR` branch extracts the method name with regex and, if it is a JUnit method, returns `STATIC IMPORT ERROR` with the exact line to add (`import static org.junit.jupiter.api.Assertions.*;`) — instead of "method does not exist in class", which led the LLM to fix the wrong problem and burn all 3 repair attempts until permanent_skip. `_auto_inject_missing_imports()` also injects the static import deterministically when it detects assertion usage without it, even before the repair loop
- **A — Constructor Type Hints**: `_extract_simplified_header()` in `context.py` now includes the type of each parameter in the `// CONSTRUCTOR CALL:` hint (e.g.: `new Foo(String account, BigDecimal amount, Long version)`), not just the name. Balanced-parenthesis reading supports annotations like `@JsonProperty("x")` in records without breaking on the inner `)`.
- **B — Expected Class in Repair Prompt**: `_build_validator_correction_prompt` in `ai/model.py` gained an `expected_class` parameter — uses the destination file name (e.g.: `MerchantCategoryCodesServiceImplTest`) instead of the production class name, preventing the LLM from continuing to abbreviate the name in the INTEGRITY ERROR repair loop
- **C — Data Holder Javadoc Skip**: `javadoc_runner.py` skips `@Document`/`@Entity`/record/enum with reason `data_holder` before any LLM call — zero tokens on classes where Javadoc adds nothing
- **D — Ollama VRAM Unload Pre-Javadoc**: `main.py` unloads `MODEL_CLEAN` via `keep_alive=0` before the JAVADOC phase — frees saturated VRAM after ~3h of test generation and prevents timeout cascades
- **E — BigDecimal→Long Handler**: specific handler in `_categorize_build_error` for `incompatible types: BigDecimal cannot be converted to Long` — returns `"Use 1L for Long literals or Long.valueOf(1)"` instead of falling back to the generic ResponseEntity handler
- **F — java.sql.Timestamp Trinity**: 3 layers for SQL format (space, not `T`): `Timestamp` in `_JDK_IMPORT_MAP` (auto-inject); preventive rule `### JAVA SQL TIMESTAMP CONSTRUCTION` when class uses `java.sql.Timestamp`; first-priority handler in `_categorize_build_error` for `IllegalArgumentException + timestamp format` returning surgical instruction
- **G — Single-Language Prompts**: `soul.md` fully translated to English — all prompts are now 100% EN, eliminating LLM language confusion (PT in soul + EN in constraints). `report_runner.py` intentionally kept in PT (report for the user, not for the LLM)
- **H — English Validator Messages**: `validate_class_name_matches_file` in `validator.py` translated to EN — messages in `failed_files.json` now consistent with the rest of the log
- **I — Stack+Reason Auto-Expire**: `_AUTO_EXPIRE_STACK_PATTERNS` expanded with post-A patterns (`actual and formal argument lists differ in length`) and post-H (`O arquivo se chama`). `FailedFilesTracker.reset()` now concatenates `stack_trace + reason` before checking patterns — `BalanceServiceImplTest` (pattern in stack_trace) and `MerchantCategoryCodesServiceImplTest` (pattern in reason) unblocked automatically
- **P0 — Deterministic Data Holder Tests**: pure classes (`@Document`/`@Entity`/POJO with only fields+ctor+getters/setters) bypass the LLM. `is_pure_data_holder()` requires literal `return field;` and `this.field = param;`. Table of supported types (String, BigDecimal, Timestamp SQL format, Long, etc.). Unsupported type → `None` → LLM fallback. **Eliminates LLM for the historically largest source of failures.**
- **P1 — Fixed Ollama Seed**: `OLLAMA_SEED=42` injected into `call_model` options makes generation reproducible run-to-run. Repair loop is not affected (changes the prompt on each attempt)
- **P2 — Per-File Commit**: `commit_single_file` in `git_utils/repo.py` called at both acceptance points of `generate_tests` (deterministic P0 + LLM). Interrupted run preserves accepted tests
- **P3 — Slim active_rules**: helper `_build_active_rules()` consolidated 12+ `### MANDATORY` blocks (IMPORTS PRESENT + SELF-IMPORT + IMPORT PROHIBITION → single `### IMPORTS`; conditionals compressed to ≤4 lines; mandatory declarations stated once). Regression test verifies 18 key phrases preserved
- **P4 — Surgical Python Patch**: `_try_surgical_patch()` applies a one-line patch for deterministic assertions (`assertNull`↔`assertEquals` + value swap) BEFORE `call_ai_with_correction`. Window ±3 lines for Maven off-by-N. Eliminates full-file regeneration for known errors
- **S4 — assertNull Mirror Handler**: handler in `_categorize_build_error` for `expected: <null> but was: <X>` when X is a real non-null value (mirror of S1) — returns instruction to replace `assertNull(expr)` with `assertEquals("X", expr)`. Preventive rule `### SETTER/GETTER TEST PATTERN` in `active_rules` when class has constructor with parameters
- **Option 1 — Pro-active Constructor Validator**: `_fix_constructor_calls(test_code, dep_context)` parses `// CONSTRUCTOR CALL: new X(Type1 p1, ...)` from dep_context and, for each `new X(args)` in the generated code whose arg count does not match the canonical signature, rewrites with sample values from the type→literal table (String→`"sampleA"`, BigDecimal→`new BigDecimal("100.00")`, Long→`1L`, Timestamp SQL format, etc.). Args splitter respects balanced parens + generics + strings. Hooked at both call sites of `generate_tests` (initial generation + repair) after `_auto_inject_missing_imports`, before `write_file`. Pre-Maven — catches the most frequent compile error (record/class constructor mismatch) without burning 3 repair attempts. Defensive posture: if a required type is not in the table, leaves the call intact
- **Option 2 — int/double→Long/BigDecimal Handler (Fix E2)**: extends the `incompatible types` branch of `_categorize_build_error` with two cases before the generic fallback — `int|short|byte → Long` returns instruction to add `L` suffix (1 → 1L); `int|double|short|byte|float → BigDecimal` returns instruction to wrap in `new BigDecimal("...")` or `BigDecimal.valueOf(...)`. Original E handler (BigDecimal→Long) preserved intact. Without this fix, these errors fell into the phantom handler that mentioned `ResponseEntity` — completely irrelevant and misleading
- **Option 5 — Fail-Fast on Irrecoverable Hallucination**: `_is_irrecoverable_hallucination(repair_hint, prod_imports)` inspects the return from `_categorize_build_error`. Returns `True` when (a) `IMPORT ERROR` names a class absent from `_prod_imports` + `_JDK_IMPORT_MAP` + known JUnit/Mockito/Spring (and is not a STATIC IMPORT that P2 fixes), OR (b) `METHOD ERROR` names a method NOT in the `_UBIQUITOUS_METHOD_NAMES` allowlist (distinguishes recoverable `getName`/`getValue` from irrecoverable `getCode`/`getFooBar`). Hooked right after `_categorize_build_error` in the repair loop: if `True`, `break` exits the loop before burning 3 attempts × ~5min. Saves ~50–70% of time on condemned files
- **Option 6 — Project-Wide Import Map**: `build_project_imports(repo_path)` scans `src/main/java/` once per run, maps `{ShortName: "import full.pkg.ShortName;"}` for every `public` class/enum/interface/record. `_auto_inject_missing_imports` (S1) gained an optional `project_imports` parameter consulted as fallback after `prod_map` and `_JDK_IMPORT_MAP`. When the LLM writes `TransactionStatusCode.APPROVED` in a controller test that does not import the enum, S1 now injects `import com.caju.transactionauthorizer.enums.TransactionStatusCode;` deterministically — no Maven failure, no repair loop. Short-name collision policy: last-seen wins (acceptable for this codebase where short names are unique)
- **Option 8 — Enum Support in P0**: `_extract_enum_constants(dep_context, enum_name)` parses `public enum X { ... }` block from dep_context (handles simple constants and constructor-args constants like `APPROVED("00")`). `generate_data_holder_test` accepts `dep_context` parameter — when a constructor field type is not in `_TYPE_SAMPLES`, tries to resolve via enum and uses the first constant as sample value (`CategoryCodeName.FOOD`). Enum import extracted from production class imports. Safe-fail: without dep_context or enum not found → returns `None` (LLM fallback as before). Unblocks data holders with enum fields (e.g.: `MerchantCategoryCodesDocument` with `CategoryCodeName`) — now generated via deterministic path instead of falling into the LLM with Mockito hallucinations
- **Option 9 — Hallucination Detection via test_code Scan**: `_is_irrecoverable_hallucination` gained optional parameters `test_code` and `project_imports`. When `repair_hint` contains `PACKAGE ERROR` AND `test_code` is passed, scans for `[A-Z]\w+\.MEMBER` — any referenced class NOT in `{prod_imports, _JDK_IMPORT_MAP, project_imports, common JUnit/Mockito/Spring}` is an irrecoverable hallucination → break from repair loop in 1 attempt instead of 3. Complements Option 6: if 6 could not inject because the class does not exist in any map, 9 prevents the LLM from inventing variations for 3 attempts
- **N7 — P4 detects object toString**: `_try_surgical_patch` case 1 (expected=null) had a bug where it always quoted `actual` as a String literal. When `actual` was an object's toString (e.g.: `TransactionCodeModel[code=null]`), it produced `assertEquals("...", expr)` which compiles but fails worse at runtime with type-mismatch. Fix: detects `[/]/=/(/)` in actual and uses `assertNotNull(expr)` instead of quoting (the only safe assertion — we know it's non-null but cannot assert the value)
- **N8 — Revert P4 patch when it worsened the error**: `_maven_output_got_worse(old_out, new_out)` compares normalized signatures (without paths/lines/values) of `[ERROR]` lines. Returns True if new output has MORE errors or new category. Safety net post-P4 in repair loop: saves `_original_code`/`_original_out`, applies patch, if Maven fails again and got worse, REVERTS `write_file` + variables before falling into LLM repair. Ensures the LLM sees the ORIGINAL error (simpler) instead of the worse one produced by the patch
- **N9 — Abort after 2 consecutive timeouts per file**: `ai/model.py` gains a module-level counter `_consecutive_timeouts`. `call_model` increments on `requests.exceptions.Timeout` and resets on success. `_try_local_agent` aborts before the next attempt if `>= _TIMEOUT_ABORT_THRESHOLD=2`. `generate_tests` calls `reset_consecutive_timeouts()` at the start of each `for main_file` iteration (per-file counter). APIs: `get_consecutive_timeouts()`, `reset_consecutive_timeouts()`. Reduces max wall-clock per difficult file from ~35min to ~15min — cuts the 14b×2 + 7b×2 = 4×420s cascade before reaching 4 full timeouts
- **N10 — Preventive controller null-input rule**: `_build_active_rules` injects `### CONTROLLER NULL-INPUT TEST PATTERN` block when class has `@RestController` AND method returns `ResponseEntity`. Explains that `response.getBody()` is NOT null by default — depends on what `service.method(null)` returns. Offers SOLUTION 1 (`when(service.method(any())).thenReturn(null)`) and SOLUTION 2 (assert on structure, not on body). Prohibits `assertNull(response.getBody())` without stub. Coexists with N4 (ResponseEntity vs @ResponseStatus) — both for controllers with ResponseEntity, covering distinct Spring gaps

## Model Hierarchy

| Role | Default model | Size | Responsibility |
|-------|--------------|---------|-----------------|
| Ultimate (SOLID) | `qwen2.5-coder:14b` | 9.0 GB | SOLID, architecture, critical review |
| Advanced (Clean) | `qwen2.5-coder:14b` | (same) | Clean code, business logic, **tests** (was `gemma4:latest`) |
| Recovery | `qwen2.5-coder:14b` | (same) | Repair loop — code-aware, code-specialized |
| Standard (Struct) | `qwen2.5-coder:7b` | 4.7 GB | Structure, naming, diff review, planner |
| Light (Doc) | `qwen2.5-coder:7b` | (same) | **Javadoc** (was `neural-chat:7b` — code-aware prevents structure changes) |

> Only **2 physical models** occupy RAM: `qwen2.5-coder:7b` (Standard/Light) and `qwen2.5-coder:14b` (Ultimate/Advanced/Recovery). All overridable via `.env`.

```bash
ollama pull qwen2.5-coder:14b   # Ultimate / Advanced / Recovery
ollama pull qwen2.5-coder:7b    # Standard / Light
```
